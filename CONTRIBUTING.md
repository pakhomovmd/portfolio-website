# Contributing to Portfolio Website

Спасибо за интерес к улучшению этого проекта! 

## Как внести вклад

### Сообщить об ошибке

Если вы нашли баг:

1. Проверьте, не был ли он уже сообщен в [Issues](https://github.com/pakhomovmd/portfolio-website/issues)
2. Если нет, создайте новый Issue с подробным описанием:
   - Шаги для воспроизведения
   - Ожидаемое поведение
   - Фактическое поведение
   - Скриншоты (если применимо)
   - Версия браузера и ОС

### Предложить улучшение

Есть идея для улучшения?

1. Создайте Issue с тегом `enhancement`
2. Опишите предлагаемую функциональность
3. Объясните, почему это будет полезно

### Pull Requests

1. **Fork репозиторий**
```bash
git clone https://github.com/pakhomovmd/portfolio-website.git
cd portfolio-website
```

2. **Создайте ветку для изменений**
```bash
git checkout -b feature/amazing-feature
```

3. **Внесите изменения**
   - Следуйте существующему стилю кода
   - Добавьте комментарии где необходимо
   - Обновите документацию если нужно

4. **Закоммитьте изменения**
```bash
git add .
git commit -m "Add: amazing feature"
```

Формат commit messages:
- `Add:` - новая функциональность
- `Fix:` - исправление бага
- `Update:` - обновление существующего функционала
- `Remove:` - удаление кода
- `Refactor:` - рефакторинг без изменения функциональности
- `Docs:` - изменения в документации

5. **Push в ваш fork**
```bash
git push origin feature/amazing-feature
```

6. **Создайте Pull Request**
   - Опишите ваши изменения
   - Укажите связанные Issues (если есть)
   - Добавьте скриншоты (если применимо)

## Стандарты кода

### TypeScript
- Используйте строгую типизацию
- Избегайте `any` типов
- Документируйте сложные типы

### React
- Используйте функциональные компоненты
- Следуйте принципам React Hooks
- Избегайте излишней вложенности

### Стилизация
- Используйте Tailwind CSS классы
- Следуйте существующей цветовой схеме
- Обеспечьте адаптивность

### Производительность
- Оптимизируйте изображения
- Используйте lazy loading где возможно
- Минимизируйте re-renders

## Тестирование

Перед отправкой PR убедитесь:

- [ ] Проект собирается без ошибок (`npm run build`)
- [ ] Нет TypeScript ошибок (`npm run lint`)
- [ ] Сайт корректно работает на desktop и mobile
- [ ] Все анимации работают плавно
- [ ] Lighthouse Score остается 95+
